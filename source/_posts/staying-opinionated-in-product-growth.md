## 引言

我曾经创建了一家公司，最终发展到超过600人的规模。

管理这样规模的公司并不容易，尤其是你需要不断重申自己是什么，不是什么。

我们经常谈论"产品劣化"现象，这在成长过快的软件产品中经常发生。

它们变得更糟，因为需要不断取悦新用户、满足新需求、处理每一个边缘情况。

这可能创造出功能强大的工具，但也可能是极其复杂的工具，有时这种复杂性会直接转嫁给用户。

## 成长的代价与选择

需要明确的是，我并不是说这种劣化不可避免。确实有一些优秀的产品成功地实现了良性成长。但人们往往没有意识到这种成长背后的困难：

{% note warning 核心观点 %}
构建产品就是做选择。
{% endnote %}

这个道理同样适用于你如何谈论自己的产品。

### 从真实到妥协的演变

在开始阶段，产品的表达天然是真实的，因为是创造者在直接交流。有时可能显得笨拙，但很直接，很有观点。

创造者会明确说出他们想要去哪里，他们是什么，他们不是什么。

然后随着时间推移，团队接管了沟通工作。你需要吸引更多人，吸引新的用户群体，传达更平滑的信息。

这时就存在一个真正的风险：

{% note danger 产品劣化警告 %}
试图取悦所有人，最终却谁也取悦不了。
{% endnote %}

## Writizzy的选择：简单而有力的宣言

我之前和Thomas讨论过Writizzy主页的设计。

它故意做得非常干净、非常简单。只有一个宣言，就这样。

对于一家年收入数百万美元的公司来说，这样做是很少见的。

因为**你需要转化率，对吧？！**

但我认为在开始阶段，你应该专注于说出你是谁。

### 宣言的力量

在我之前的公司，我们在第一年就有一个宣言。

那个宣言阐述了我们是谁，我们想要做什么。

> 你可以自由地自由工作

来到网站的人不需要滚动5分钟浏览几十个精美设计的模块。事实是，对于一个你不了解的产品，没有人会这样做。

新访客需要在不到10秒内决定是否继续。需要有一个强有力的声明来吸引他们。

对我们来说，这个声明就是我们是谁，以及我们不是谁。

## 负面表达的勇气

**一个不浪费你时间的博客平台。**

这个表述并不积极。有人向我们指出了这一点。为什么不直接说"一个为你节省时间的博客平台"？

确实，那样更容易被接受。但我们刚刚出现，不可避免地会被拿来与其他产品比较。这很自然。

我们到底在这里做什么？

我们感到沮丧，我们在构建这个产品来对抗某些东西。

我们想要创造一个简单的产品。

但不是简陋的产品。

### 我们的使命

除此之外，我们想要让人们重新获得轻松写作的能力，而不是被迫在职业网络上炫耀，或者在充满机器人的社交媒体平台上制造虚假互动。

也许明天我们的主页会充满彩色模块和华丽的营销口号。

## 产品劣化的更广泛定义

需要澄清的是，我这里谈论的不仅仅是传统意义上的"enshittification"。我使用的是更广泛的定义，指的是由不断增长的复杂性驱动的产品衰退。

产品衰退的方式不只一种：

### 常见的衰退模式

{% note info 产品衰退的四种模式 %}
1. **功能膨胀**：为了满足所有用户需求而添加过多功能
2. **界面复杂化**：试图在一个界面中展示所有功能
3. **消息稀释**：为了吸引更多用户而模糊产品定位
4. **核心价值迷失**：在成长过程中偏离最初的产品愿景
{% endnote %}

## 如何在成长中保持初心

{% note success 保持初心的四个策略 %}
### 1. 明确你的"不是什么"
定义产品边界和你拒绝做的事情，往往比列出功能清单更重要。

### 2. 保持宣言的简洁性
一个强有力的宣言胜过千言万语的功能介绍。

### 3. 勇于表达观点
不要害怕得罪一些人，因为试图取悦所有人的产品往往平庸无奇。

### 4. 定期回顾初心
随着团队和产品的成长，定期回顾最初的愿景和价值观。
{% endnote %}

## 对创业者的启示

### 早期阶段的策略

- **专注身份认同**：清楚地表达你是谁，而不是你能做什么
- **接受争议**：有观点的产品会引起争议，这是好事
- **简化信息**：10秒钟内让用户明白你的价值主张

### 成长阶段的挑战

- **抵制功能膨胀**：不是每个用户请求都需要满足
- **保持沟通一致性**：即使团队扩大，也要保持品牌声音的一致性
- **定期做减法**：移除不符合核心愿景的功能

## 结语

在产品成长的过程中，最大的挑战不是技术或资金，而是保持初心。

当你试图取悦所有人时，你往往会失去最初吸引用户的那些特质。

真正成功的产品不是没有缺点的产品，而是在某些方面做到极致，同时勇于承认自己不适合某些用户的产品。

记住：**构建产品就是做选择。**选择你要服务的用户，选择你要解决的问题，也选择你要拒绝的诱惑。

只有这样，你的产品才能在成长中保持灵魂，而不是变成一个试图满足所有人却让所有人都不满意的平庸工具。
